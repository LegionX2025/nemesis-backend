# MASTER INSTRUCTIONS SYSTEM - GEMINI 3.1 PRO DEEP-THINKING

## AUTO-AUDIT PROTOCOL
1. Enforce strict separation of concerns across Frontend, Backend, and Workflows.
2. Validate frontend rendering pipelines, backend fetching mechanisms, and data processing.
3. Automatically monitor API executions and trace timelines.
4. Ensure no hardcoded secrets (Neo4j, MongoDB, Cloudflare) exist in prompts or code. All credentials MUST be loaded dynamically from environment variables.

## SECURE CREDENTIAL MANAGEMENT
- All database and API keys must be referenced via environment variables (e.g. `${NEO4J_URI}`, `${MONGODB_URI}`, `${CLOUDFLARE_ACCOUNT_ID}`).
- Ensure live credentials are rotated if ever exposed and stored in a secure secrets manager (like Cloudflare Secrets).

## 1. LOGO & ASSET RESOLUTION (No Hardcoded Assets)
Do not manually maintain thousands of icons. The Global Asset Resolver automatically discovers, verifies, caches, and updates logos.

### Resolution Priority:
1. cryptologos.cc (Primary for Chains, Coins, Tokens, Stablecoins)
2. CoinGecko Assets
3. Trust Wallet Assets
4. DefiLlama Assets
5. CoinMarketCap (metadata only)
6. Official Project Website / Explorer Favicons / Local Cache

### Supported Entities (Local Assets):
Entity logos must be maintained in a local asset pack (`assets/entities/`): Binance, Coinbase, Kraken, OKX, Bybit, Uniswap, Tornado Cash, etc.

## 2. AUTONOMOUS TRACE EXECUTION (WebSockets)
- DO NOT use polling for trace operations.
- Use WebSockets for real-time trace execution and progressive UI updates.
- The UI should stream events immediately: `Scanning Ethereum...` -> `Resolving Entities...` -> `Building Graph...`
- Provide instant feedback instead of blocking HTTP requests.

## 3. DEEP TRACE VISUALIZATION
The knowledge graph must grow in real time. Nodes and edges must animate progressively as the background workers process hops, cross-chain bridges, and mixers.

## 4. BACKGROUND WORKERS
Never perform long-running traces inside the HTTP request.
Architecture Flow: `FastAPI` -> `Job Queue` -> `Trace/Spider/Graph Workers` -> `WebSocket Updates`

## 5. DATABASE STRATEGY
### Cloudflare D1 (Lightweight / Fast)
- Store: Wallet Labels, Entity Labels, Explorer Metadata, Protocol Metadata, Chain Registry, Token Registry, Logo Cache, Classification/Risk Rules.

### MongoDB / Neo4j (Heavy Lifting)
- Store: Trace Jobs, Graph Data, Hop Trees, Evidence, Transaction Cache, Reports, Risk Scores, Wallet Profiles.
- D1 must remain fast and compact. Do not store large trace graphs in D1.

## 6. GLOBAL IDENTITY RESOLUTION LAYER (GIRL)
Every object rendered in the UI must pass through the Global Identity Resolution Layer to become a standardized, rich intelligence object.
- Never display a plain address, token symbol, or chain name without its corresponding metadata, logo, and classification.

### Rendering Priority:
Wallet Address -> Resolve Entity -> [Entity Logo, Classification Icon, Chain Logo, Risk Badge]

### Standard UI Component Layouts:
**Wallet UI:**
[Chain Logo] [Entity Logo] Wallet Address
Entity Name | Classification | Risk Badge | Verification Link

**Transaction UI:**
[Chain Logo] [Native Token Logo] [Transferred Token Logo] Amount | USD Value | Timestamp | Explorer Link

**Graph Nodes:**
[Entity Logo] [Chain Logo] [Classification Badge]

## 7. AUTOMATIC ENTITY RESOLUTION PIPELINE
Wallet -> Explorer -> Page Title -> Explorer Labels -> Deep DOM -> JS State -> Graph Correlation -> Behavior -> Swarm Consensus -> Resolved Entity.

## 8. INTELLIGENCE TAGS & CLASSIFICATION
Entities must accumulate normalized tags:
- **Behavioral:** High Velocity, Wash Trading, MEV
- **Infrastructure:** Exchange, Custodian, Bridge, Mixer
- **Compliance:** OFAC, Sanctioned, Scam
- **Technical:** EOA, Smart Contract, Proxy Contract

## PRODUCTION DEPLOYMENT RECOMMENDATION
Implement a centralized Global Asset Registry. Every UI component, graph node, and timeline event must query this registry before rendering to ensure all data conforms to the canonical JSON schema, ensuring consistent identity and iconography regardless of the underlying blockchain.
