# Confidence Formula

Instead of a simple average, use weighted evidence:

```
Confidence =
Σ(Evidence Weight × Evidence Reliability × Freshness × Source Trust × Correlation Strength)
────────────────────────────────────────────────────────────────────────────────────────────
Σ(Max Possible Weight)
```

Evidence weights can prioritize:

* On-chain cryptographic evidence
* Regulatory filings
* Court documents
* Exchange attribution
* Verified OSINT
* Commercial intelligence
* User-submitted intelligence (lowest default weight)

# Real-World Attribution Probability

A Bayesian approach is well suited:

```
P(Person | Evidence)

=
P(Evidence | Person)
× P(Person)
÷ P(Evidence)
```

Evidence can include:

* ENS ownership
* GitHub commits
* DAO governance votes
* Domain ownership
* Exchange deposit behavior
* Corporate treasury interactions
* Public legal records
* Social profile linkage
* Repeated behavioral fingerprints

# Asset Obfuscation Score

A composite score (0–100):

```
Obfuscation Score =

(Bridge Weight × Bridge Count)
+
(Mixer Weight × Mixer Exposure)
+
(Cross-Chain Weight × Chain Hops)
+
(Time Delay Weight × Delay Entropy)
+
(Split Weight × Fan-Out Factor)
+
(Peel Weight × Peel Depth)
+
(Privacy Weight × Privacy Tool Usage)
```

Interpretation:

* **0–20:** Minimal obfuscation
* **21–50:** Moderate complexity
* **51–80:** High obfuscation
* **81–100:** Sophisticated laundering pattern
