2. NEMESIS Apps, Frontends & Backends

NEMESIS Tracer

Frontend: nemesis_tracer.html, tracer_app/ (React)

Backend: backend/app/main.py, backend/nemesis_apex_omni.py

Purpose: Visualizing blockchain traces, crypto flows, and OSINT correlations.

NEMESIS ID

Frontend: nemesis_id.html, nemesis_id_new.html

Backend: /api/nemesis_id/* endpoints

Purpose: Individual entity profiling, KYC/AML scoring, and AI-driven insights.

NEMESIS Apex Omni

Frontend: apex_dashboard.html, apex_landing.html

Backend: backend/nemesis_apex_omni.py

Purpose: Enterprise-level threat hunting, global blockchain tracking, ransomware analysis.

NEMESIS Threat Hunter / Darknet

Frontend: nemesis_threat_hunter.html, darknet_search.html

Backend: /api/osint/* endpoints

Purpose: Scanning dark web forums, identifying exploiters and ransomware affiliates.

3. API Routes & Endpoints

Endpoint Method App / Service Purpose
/api/v1/trace/deploy POST Apex Omni Deploys a tracer agent on a specific target address.
/api/v1/ransomware/hunt POST Apex Omni Initiates global threat hunt for ransomware strains.
/api/entity POST/GET GBIO Ontology Add or retrieve entity metadata (Graph Node).
/api/nemesis_id/profile/{address} GET Nemesis ID Fetches comprehensive KYC/AML profile for an address.
/api/nemesis_id/tx_history/{address} GET Nemesis ID Retrieves categorized transaction history.
/api/nemesis_id/ai_insights/{address} GET Nemesis ID / AI Generates LLM-based behavioral insights on a wallet.
/api/osint/{address} GET Threat Hunter Pulls Clear/Dark web intelligence correlated to an address.
/api/start_trace POST/GET Tracer App Kicks off the Graph extraction from blockchain nodes.
/api/ai/fabric_status GET System Returns health of LLM integration (Gemini/OpenAI).

5. Full High-End Architecture & Executions

The NEMESIS architecture relies on a highly decoupled event-driven system leveraging FastAPI, Celery Workers, and WebSockets (Socket.IO) for real-time trace streaming.

Execution Workflow Diagram

sequenceDiagram participant User as Analyst UI participant API as FastAPI Backend participant GBIO as GBIO Ontology Engine participant OSINT as Threat Intel Module participant DB as Graph DB (Neo4j/Mongo) User->>API: POST /api/start_trace (Target Wallet) API->>GBIO: Initiate Trace & Classify GBIO->>DB: Fetch Known Entity Labels GBIO->>OSINT: Query Darknet/Clearweb Data OSINT-->>GBIO: Return Threat Signals & Sanctions GBIO->>GBIO: Execute Topological AML Profiling GBIO-->>API: Stream Nodes & Edges (WebSockets) API-->>User: Visual Graph Rendered in Real-Time
